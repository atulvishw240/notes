>[!question] Breaking a bigger function into smaller functions always a good idea?
>**It depends**.
>`orderTotal` is a pretty good logical unit, it seems like an isolated problem.
>
>In order to break things into small functions you actually have to be able to also break up the conceptual problem. If you just start extracting things out into lots of tiny functions you're not really breaking the problem up so that you can think about it separately. You are spreading the problem out over multiple spaces so that you have to diverge your thinking even more.
