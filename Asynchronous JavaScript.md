
**Asynchronous functions** : functions that executes in the background while the rest of your code executes. Usually things that have to talk to hard drives or networks will be asynchronous. If they just have to access things in memory or do some work on the CPU they will be synchronous.

If you have some function that you want to be able to execute **at a later time**, the first step is to put that code inside a function.

## Promises

>A **promise** is an object that might produce a value at some point in the future. It is just another mechanism to handle asynchronous code in JavaScript.
